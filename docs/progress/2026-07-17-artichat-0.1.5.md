## 2026-07-18 Update Settings and Announcement

- Removed the Version panel from General settings and made `Settings > Update` the final settings item.
- Rebuilt the version panel as an ArtiChat card with current version, build hash, running status, stable-release status, manual check, deploy confirmation, progress polling, retry state, and expandable release notes.
- Removed the global update toast from the application layout. Version checking and deployment now live only in the admin Update page.
- Added a single announcement card below the version card. It displays only the remote `type`, `published_at`, `title`, and plain-text `content`; there is no detail link and remote HTML is never rendered.
- Fixed the official announcement source to `https://artichatupdate.artivis.cc/index.json`; deployments cannot override this URL. `ARTICHAT_ANNOUNCEMENT_CACHE_TTL_SECONDS` remains configurable (default 600), as does `ARTICHAT_ANNOUNCEMENT_TIMEOUT_SECONDS` (default 4).
- Added admin endpoint `GET /api/v1/updates/announcement`. The backend fetches the configured JSON object, validates field lengths and types, filters expired `expires_at`, limits responses to 64 KiB, caches one result, and falls back to the last good result on transient failures.
- Expected remote shape is a single JSON object, for example:

  ```json
  {
    "id": "1",
    "title": "ArtiChat公告",
    "content": "公告正文",
    "type": "info",
    "published_at": "2026-07-18T10:00:00+08:00",
    "expires_at": null
  }
  ```

- The official endpoint was verified on 2026-07-18 with HTTP 200 and valid JSON. Its current sample object renders as a single announcement card in the live acceptance container.
