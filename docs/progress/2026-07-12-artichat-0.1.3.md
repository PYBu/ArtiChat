## Milestone 2: Four-Part Pricing And Usage Ledger

- Status: implementation, migration rehearsal, and automated regression complete; final browser acceptance remains in the consolidated local test stage after milestone 5.
- Model policies now define input, output, cache-creation, and cache-read prices in `Chatpoint / 1M Token`.
- Legacy `usage_multiplier` remains compatible and maps to equivalent input/output prices at `100 * multiplier`; cache prices start at zero.
- Provider usage is normalized into four non-overlapping Token categories. OpenAI cached prompt subsets are removed from normal input while Anthropic cache buckets remain separate.
- Metered balance debit, balance ledger insertion, and usage insertion use one database commit and roll back together on failure.
- New usage records snapshot all four prices, raw provider usage, request ID, first-content latency, total duration, status, and audit IP.
- User usage responses use an explicit safe projection and cannot serialize IP, raw usage, internal metadata, or another user summary.
- Admin usage supports user ID, model, status, and date filters and displays full IP audit data.
