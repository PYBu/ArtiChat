# ArtiChat 0.1.4 Progress

## Task 1: Verification Code Modal Flow

- Status: implementation and local acceptance complete; awaiting user acceptance at `http://localhost:3000`.
- Registration now collects the name, email, password, and optional password confirmation before requesting a registration code. A successful request opens a dedicated verification modal instead of adding a code field to the registration form.
- Email-code login now collects the email first and opens the same verification modal after the user clicks login.
- Password changes, billing-information changes, and the two-stage login-email change flow now use the shared verification modal instead of inline code fields.
- The shared modal owns the 60-second resend cooldown, 10-minute expiry display, one-time-code autocomplete, explicit validation messages, and responsive desktop/mobile layout.
- Closing a still-valid challenge and submitting the parent action again reopens and focuses the existing modal without requesting another email.
- Verification input no longer uses the browser `pattern="[0-9]{6}"` constraint. Input is normalized with Unicode NFKC, non-digits are removed, and the result is limited to six ASCII digits. Full-width and separated input such as `１２３ ４５６` is submitted as `123456`.
- The two-stage email-change flow preserves the existing security boundary. Resending after the new-email stage restarts current-email verification because the current-email grant is one-use.
- No backend API, database schema, production service, or production data was changed.
