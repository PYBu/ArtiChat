# ArtiChat 0.1.6

0.1.6 重点改善图像编辑链路与礼品卡管理体验。

## 更新内容

- **图像编辑引用修复**：上传图片和生成图片可以通过文件 ID、ArtiChat 文件地址或完整 Data URL 继续编辑；内部受保护文件由后端鉴权读取并以图片字节上传至编辑模型。
- **当前附件自动传递**：编辑工具可以自动使用当前对话中的图片附件，生成图片也会保留稳定文件 ID，避免模型处理大段 Base64。
- **多编辑引擎统一**：OpenAI、Gemini 和 ComfyUI 共用安全图片解析流程，并保留各自正确的文件格式和 MIME 类型。
- **图片地址格式**：管理员可选择相对或绝对输出地址，默认使用不依赖域名的相对地址；文件访问权限保持不变。
- **礼品卡用户选择**：发放礼品卡时可按用户名称或邮箱搜索并选择多个用户，系统仍以稳定用户 ID 执行发放。
- **用户 ID 展示**：管理员编辑用户时可直接查看并选取完整用户 ID。
- **默认公告清理**：移除系统自动插入的历史默认公告，同时保留管理员创建的公告及其状态。

## 兼容与安全

- 内部图片地址不会直接交给外部模型访问，外部模型收到的是经过当前用户权限校验后的图片文件内容。
- 外部 HTTP(S) 图片继续经过 SSRF 防护；无 Data URL 前缀的裸 Base64 会被拒绝。
- 图像编辑提供商的鉴权失败会明确提示检查 Image Edit Base URL 与 API Key。
- 公告迁移只清理 `created_by=system` 的默认内容，不删除管理员公告。
- 本版本不新增 Python、系统或 npm 依赖。
