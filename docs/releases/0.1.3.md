# ArtiChat 0.1.3

0.1.3 是一次覆盖平台管理、订阅计费、账号安全与邮件体验的系统升级。本版本补齐了邮箱验证与账号恢复闭环，加入品牌化 HTML 邮件，并继续优化管理端、移动端和 Docker 构建体验。

## 本次重点

- **完整的邮箱与账号安全流程**：支持注册验证码、邮箱验证码登录、忘记密码、一次性重置链接、登录邮箱变更，以及敏感操作前的邮箱验证。
- **品牌化 HTML 邮件**：邮件模板支持安全 HTML 编辑和实时预览；默认模板采用包含平台 Logo、平台名称与版本页脚的 ArtiChat 卡片布局，并保留纯文本备用内容。
- **更完整的订阅与用量管理**：订阅计划、模型权限、兑换码、礼品、公告和用量记录使用独立管理页面，并支持移动端导航。
- **更准确的计费与审计**：模型价格细分为输入、输出、缓存创建和缓存读取四类 Token 价格，用量总计按完整筛选条件聚合。
- **可配置的品牌信息**：管理员可以配置平台名称、关于页面内容以及浅色和深色 Logo。

## 管理与订阅

- 用户管理可直接调整订阅、有效期、Plan Chatpoint、Check Chatpoint 和管理员备注。
- 管理员可批量配置模型订阅权限，管理兑换码与礼品，并按用户、模型、状态和日期筛选用量。
- 用量记录新增请求耗时、首字延迟、客户端信息和原始用量等审计字段。
- 有待领取礼品时，侧边栏会显示入口；领取后状态会立即刷新。
- 公告删除历史默认隐藏；0.1.3 的站内更新公告会在升级后向每位用户展示一次。

## 注册、邮件与账户安全

- 支持注册邮箱域名白名单和可选的子域名匹配。
- 新增 SMTP 设置、连接测试、邮件模板管理、发送记录和普通失败邮件重试。
- 邮件验证码包含冷却时间、频率限制、有效期和失败次数限制。
- 密码重置和登录邮箱变更会撤销旧会话；已使用的验证码、重置链接和安全授权无法重放。
- 密码、支付信息、邮箱和订阅发生变更时，可发送对应的安全通知。

## 性能与稳定性

- Chatpoint 扣费、余额流水和用量记录使用同一事务，降低部分写入风险。
- 邮件模板与订阅计划的默认数据仅在实际变化时写入，减少重复查询和空提交。
- 批量生成兑换码改为一次数据库重复检查，并限制单次批量与管理列表大小。
- 公告、订阅、更新和邮件 API 使用统一的认证与错误处理；邮件预览会取消过期请求，避免快速切换模板时显示旧响应。
- Docker 构建上下文排除本地构建、缓存和备份目录；Pyodide 版本与锁文件保持一致，重复构建不再误删缓存。

## 兼容性与升级提示

- 本版本包含数据库迁移。升级前请保留现有数据备份，并等待迁移完成后再开始使用。
- 启用注册验证、邮箱登录、密码重置或敏感操作验证前，请先完成 SMTP 配置并发送测试邮件。
- 默认 Docker Compose 现在只运行 ArtiChat，不再内置启动 Ollama。Ollama 接入能力保持不变，请通过 `OLLAMA_BASE_URL` 连接主机或其他服务器上已有的 Ollama 服务。
- 旧版 `usage_multiplier` 配置继续兼容，升级后无需立即重配已有模型。
- 生产环境应设置并长期保留稳定的 `WEBUI_SECRET_KEY`，避免会话与敏感配置在容器重建后失效。
