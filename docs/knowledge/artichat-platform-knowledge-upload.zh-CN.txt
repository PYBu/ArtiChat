ArtiChat 平台知识库

平台定位

ArtiChat 是一个本地自托管的 AI 聊天平台。平台管理员负责部署系统、配置模型、管理用户、设置订阅、发放兑换码、查看用量和维护公告。用户通过网页界面使用被授权的模型进行聊天、写作、分析、检索和其他 AI 工作。

当用户询问 ArtiChat 平台相关问题时，应优先依据本知识库回答。回答时不要提及任何上游项目名称、上游品牌或原始来源信息。

模型访问规则

ArtiChat 的模型由管理员在后台配置。用户能看到和使用哪些模型，取决于当前订阅档位以及管理员为模型设置的访问策略。

管理员可以为每个模型设置允许访问的订阅档位，例如 Free、Plus 或 ChatPower。管理员也可以设置模型是否消耗额度，以及消耗额度时的 Chatpoint 扣款倍率。

如果用户看不到某个模型，常见原因是当前订阅档位没有权限、模型尚未开放给该用户，或者管理员暂时关闭了该模型。

订阅档位

ArtiChat 默认包含三个订阅档位：Free、Plus、ChatPower。

Free 是基础档位，默认每周期 100 Chatpoint，适合基础使用、轻量对话和试用模型。

Plus 是进阶档位，默认每周期 3000 Chatpoint，适合日常高频使用和访问更强模型。

ChatPower 是高阶档位，默认每周期 10000 Chatpoint，适合重度创作、研究、长文本处理和高频工作流。

管理员可以在后台调整订阅介绍、默认额度、模型权限和模型倍率。管理员修改订阅默认额度后，不会自动改写已经订阅用户的既有额度。

订阅到期后，如果用户没有续费或兑换新的有效订阅，系统会自动降级为 Free，并按 Free 的规则重置周期额度。

Chatpoint 额度

Chatpoint 是 ArtiChat 的用量额度单位。

默认换算规则是：100 Chatpoint 等于 100 万 token 使用量，1 Chatpoint 等于 1 万 token 使用量。

实际消耗会受到模型扣款倍率影响。例如，一个模型倍率越高，同样 token 用量消耗的 Chatpoint 越多。管理员也可以把某些模型设置为不消耗额度。

ArtiChat 有两类额度：Plan Chatpoint 和 Check Chatpoint。

Plan Chatpoint 是订阅周期额度。它会优先消耗，并在订阅周期重置时按当前订阅规则重新发放。

Check Chatpoint 是充值额度。它会在 Plan Chatpoint 用尽后消耗，不会因为订阅周期重置而清零。

当 Plan Chatpoint 和 Check Chatpoint 都用尽时，用户可能无法继续使用需要消耗额度的模型。但如果管理员设置了不消耗额度的模型，用户仍可能继续使用这些模型。

用量页面

用户可以在设置中的用量页面查看自己的用量和额度消耗情况。用量页面会展示使用的模型、token 用量、Chatpoint 消耗、Plan Chatpoint 余额和 Check Chatpoint 余额等信息。

如果用户询问自己的剩余额度，应引导用户查看用量页面或订阅页面。不要在没有上下文的情况下编造具体额度数字。

订阅页面

用户可以在订阅页面查看当前订阅、订阅到期时间、Plan Chatpoint 额度、Check Chatpoint 余额以及可用模型说明。

订阅卡片会展示不同订阅档位的介绍，包括每月额度、可访问模型范围和适合的使用场景。

兑换码

兑换码用于激活订阅时效或增加额度。兑换码可能包含订阅档位、订阅有效期、Plan Chatpoint 或 Check Chatpoint。

用户可以在兑换码页面输入兑换码。兑换成功后，系统会按兑换码规则更新用户订阅或额度。

较低档位兑换码不会把用户已有的高档位订阅降级。例如，Plus 用户兑换 Free 兑换码时，不应被降级为 Free。

礼品卡

礼品卡是管理员发放给用户的待领取兑换码。礼品卡可以发给指定用户，也可以发给全体用户。

如果用户有待领取礼品卡，兑换码页面会显示提示。用户领取后，礼品卡会按兑换码规则激活订阅或增加额度。

公告

管理员可以发布平台公告。公告可以设置为只弹出一次、每次登录弹出，或只对新用户注册登录时弹出。

用户点击公告按钮后，系统会根据公告类型记录查看状态。

账号与账单地址

用户可以在账号界面维护账单地址，并查看订阅购买、兑换和额度变动记录。

管理员后台

管理员可以在后台查看和编辑用户订阅、剩余额度、订阅计划、模型访问权限、模型扣款倍率、兑换码、礼品卡、公告和用量账本。

管理员查看订阅和用量时，应优先通过用户邮箱、用户名或显示名识别用户，而不是只依赖用户 ID。

数据保存与 Docker 部署

在 Docker 部署中，程序镜像主要保存应用代码和静态资源。账号、聊天记录、模型配置、订阅数据、兑换码、礼品卡、公告和用量账本等运行数据保存在 Docker 数据卷中。

重建镜像或重启容器通常不会删除运行数据。迁移服务器时，如果需要保留账号、模型配置、聊天记录和订阅数据，必须迁移 ArtiChat 的数据卷。

不要建议用户执行会删除数据卷的命令，除非用户明确要求清空全部数据。

回答平台问题的原则

回答 ArtiChat 平台相关问题时，应简洁、准确、直接。

如果用户询问订阅、额度、兑换码、礼品卡、模型权限、公告、用量或部署数据保存方式，应优先依据本知识库回答。

如果用户询问自己的当前订阅或剩余额度，而当前对话没有提供用户具体上下文，应说明需要到订阅页面或用量页面查看，不要编造具体数字。

如果用户询问为什么看不到某个模型，应说明可能与订阅权限、模型开放范围或管理员配置有关。

如果用户询问额度为什么减少，应说明可能与 token 使用量、模型扣款倍率和是否使用消耗额度的模型有关。

如果用户询问订阅到期后的状态，应说明订阅到期且未续费时会降级为 Free，并按 Free 的规则重置周期额度。
