# ArtiChat 0.1.3 Milestone 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Execution constraint:** The ArtiChat handoff forbids subagents, so execute inline with `superpowers:executing-plans`. The user authorized automatic continuation and requires local acceptance before any GitHub push.

**Goal:** Replace multiplier billing with four OpenAI-style Token rates, persist an auditable request-level usage ledger, and expose privacy-safe admin and user views.

**Architecture:** Keep model pricing in `meta.subscription`, retain `usage_multiplier` only for backward compatibility, and normalize provider usage into four non-overlapping Token buckets before billing. Add nullable audit columns for historical compatibility, snapshot all four rates on each new usage row, and commit balance debit, balance ledger, and usage row in one database transaction. The admin API returns full audit data including IP; the user API uses an explicit projection that cannot serialize IP.

**Tech Stack:** FastAPI, Pydantic v2, SQLAlchemy async, Alembic, Decimal fixed-point arithmetic, pytest/pytest-asyncio, SvelteKit, TypeScript, Node source guards.

- [ ] **Step 3: Implement Decimal four-rate billing**

Add `TOKENS_PER_MILLION = 1_000_000` and implement:

```python
def calculate_token_cost_micros(
    *,
    input_tokens: int,
    output_tokens: int,
    cache_creation_tokens: int,
    cache_read_tokens: int,
    input_chatpoint_per_million: str | Decimal | int,
    output_chatpoint_per_million: str | Decimal | int,
    cache_creation_chatpoint_per_million: str | Decimal | int,
    cache_read_chatpoint_per_million: str | Decimal | int,
) -> int:
    token_counts = [input_tokens, output_tokens, cache_creation_tokens, cache_read_tokens]
    prices = [
        Decimal(str(input_chatpoint_per_million)),
        Decimal(str(output_chatpoint_per_million)),
        Decimal(str(cache_creation_chatpoint_per_million)),
        Decimal(str(cache_read_chatpoint_per_million)),
    ]
    if any(value < 0 for value in token_counts):
        raise ValueError('token counts must be greater than or equal to 0')
    if any(value < 0 for value in prices):
        raise ValueError('token prices must be greater than or equal to 0')
    raw_chatpoints = sum(Decimal(tokens) * price for tokens, price in zip(token_counts, prices)) / Decimal(
        TOKENS_PER_MILLION
    )
    return chatpoint_to_micros(raw_chatpoints)
```

Keep `calculate_cost_micros` for compatibility tests, but stop using it in new billing code.

## Plan Self-Review

- Four non-negative prices use decimal strings and are snapshotted per request.
- OpenAI cached input is subtracted only when provider detail proves it is included; Anthropic cache buckets remain separate.
- Old `usage_multiplier` policies migrate to equivalent `100 * multiplier` input/output prices and zero cache prices.
- Usage rows retain raw provider usage and nullable historical audit fields.
- Balance debit, balance ledger, and usage record share one transaction.
- Admin bypass, unlimited, and missing usage record zero cost without losing Token or latency data.
- Admin receives IP; the user projection cannot serialize IP.
- Forwarded addresses are trusted only when `TRUSTED_PROXY_IPS` is explicitly configured.
- Admin and user interfaces show the requested Token, cost, latency, time, and status fields.
- No GitHub push, version bump, or production operation is included.
