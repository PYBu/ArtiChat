# ArtiChat 0.1.3 Milestone 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Execution constraint:** Execute inline because the project handoff forbids subagents. Continue without waiting for milestone approval, but do not push GitHub.

**Goal:** Merge subscription and quota administration into the existing user workflow and add configurable platform identity, theme logos, and About content.

**Architecture:** Enrich the admin user response with a subscription summary while retaining the existing subscription compatibility API. The edit-user modal saves profile and subscription data as one user action. Store platform text settings in the existing per-key Config table, store validated raster logo files in the persistent data directory, and expose only public branding values through the normal frontend config response.

**Tech Stack:** FastAPI, Pydantic, SQLAlchemy async, Pillow image validation, SvelteKit, existing Config table and stores, pytest, Node source guards.
